# Статус и планы — Canon Print Bridge

Дата последнего обновления: **2026-07-01**.

## Текущий статус: РАБОТАЕТ ✅

Сквозной автономный конвейер подтверждён на реальной бумаге, включая «холодный» цикл
(полное выключение XP → старт → автологин → автозапуск сторожа → автозахват USB → печать).

Что подтверждено:
- **Win11 → XP → CAPT** печать идёт целиком автоматически: выбрал PDF + параметры → «Печать» → лист.
- **USB-фильтр** VirtualBox сам захватывает принтер при старте VM (ручной выбор в меню USB не нужен).
- **Сторож** (`watcher.vbs`) автозапускается в XP, **автологин** под `User` работает.
- **Лаунчер** «Запустить принтер (VM)» поднимает VM (починен, см. ниже).
- В приложении есть **прогресс-полоса + таймер** («Печать… 0:03» → «Готово за 0:03»).

## Планы (потом)

### 1. Self-contained / переносимое решение «из коробки»
Цель: перенести на другую машину **приложение + VM `Microelectronics`** и запустить тот же
принтер, воткнутый в новую машину, **без доп. настройки**.

Что уже переносится «внутри» VM (в образе): SumatraPDF, очередь `Canon LBP-1120 A4`,
сторож в автозагрузке, автологин, USB-фильтр (он в настройках VM). Драйвер CAPT новой машине
на хосте НЕ нужен — он живёт в госте; принтер той же модели захватится фильтром по 04a9:262b.

Что машинно-зависимо и требует решения:
- **Образ VM**: экспортировать в `.ova` (`VBoxManage export Microelectronics -o ...`) для переноса.
- **Хостовый путь общей папки** (`C:\Virtualization\Shared`) задан в настройках VM и в
  `appsettings.json` → на новой машине либо воссоздать тот же путь, либо параметризовать.
  Гостевая сторона (`\\vboxsvr\Shared\Queue`) стабильна — зависит только от имени шары `Shared`.
- **Пути в конфиге** (`QueueRoot`, `LauncherPath`) — сделать относительными / автоопределяемыми
  или вынести в первый запуск (мастер настройки).
- **Бутстрап**: маленький установщик/скрипт, который на новой машине: ставит VirtualBox (или
  проверяет), импортирует `.ova`, создаёт общую папку, прописывает пути. Тогда — «из коробки».

### 2. Один exe (single-file)
- Собрать `dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true`
  (+ `-p:IncludeNativeLibrariesForSelfExtract=true`). Дать опцию и framework-dependent (меньше размер).
- Учесть `appsettings.json`: либо встроить дефолты (уже есть в `AppConfig`), либо класть рядом.

### 3. Более удобный UI
- Идеи: drag-drop крупной зоной, превью имени файла, индикатор «VM готова» (через heartbeat
  от сторожа — сторож пишет `Queue\status\watcher.alive`, приложение это читает и гасит/зажигает
  готовность кнопки печати точно, а не по таймеру).
- Возможно: история заданий, повтор последнего, выбор нескольких PDF, запоминание параметров.
- Возможно: иконка приложения, сворачивание в трей, авто-запуск VM при старте приложения.

### Прочее (мелочи)
- Часы в гостевой XP сбиты (`updatedAt` в статусах показывает прошлые даты) — синхронизировать
  время в XP, если захочется честных таймстемпов. На работу не влияет.
- Очереди `Canon LBP-1120 A5` / `B5` — создать в XP при необходимости (см. `xp-watcher/README-XP-setup.md`).
